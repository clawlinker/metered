# Landing Page Research — Directory & Marketplace Patterns

**Researcher:** Designer (Metered)  
**Date:** 2026-03-28 02:11 UTC  
**Purpose:** Analyze best-in-class directory/marketplace landing pages for value prop, trust signals, and content patterns

---

## Key Directories Analyzed

### 1. Stackshare
**URL:** https://stackshare.io  
**Strengths:**
- Hero section is crystal clear: "The place to share your tech stack" with simple form below
- Trust badges visible but NOT in hero — in sidebar and footer only
- Directory cards are dense with tech logos — immediate visual recognition
- Social proof: "Used by 250K+ developers" prominent
- Bottom of page has company logos (not brands in hero)

**Patterns to adopt:**
- Hero text should be 1 sentencemax, no tech jargon
- Trust badges separate from hero section
- Cards should be logo-dense for quick scanning

### 2. AlternativeTo
**URL:** https://alternative.to  
**Strengths:**
- Simple, clean hero with clear value prop
- Categories navigation above fold — users know they can browse
- No excessive branding in hero — clean focus on utility
- Footer has clear "About" and "Contact" links

**Patterns to adopt:**
- Hero = utility first, branding second
- Categories must be accessible immediately

### 3. G2 (for contrast)
**URL:** https://g2.com  
**Strengths:**
- Massive trust signals (review count, star ratings)
- Strong category navigation
- User-generated content dominance

**Patterns to adopt:**
- Social proof visible but not overwhelming
- Category navigation critical

### 4. Product Hunt (directory style)
**URL:** https://www.producthunt.com  
**Strengths:**
- Hero = "Discover the latest product launches" — clear purpose
- Card-based layout with upvote counts
- Minimalist design — no distractions

**Patterns to adopt:**
- Clear purpose statement in hero
- Card-based directory grid
- Minimalist, no distractions

---

## Key Findings for Metered

### 1. Hero Section Should Be
- **Headline:** 1 sentence max. "The x402 API directory for AI agents and developers"
- **No trust badges** (Coinbase, World Chain) — those go in footer
- **No feature dump** — just utility and audience
- **Clear CTA:** "Browse services" or "Submit your API"

### 2. Trust Signals
- **Hero:** None — trust comes from content quality, not logos
- **Trust cards section:** Subtle explanations of x402 + World ID
- **Footer:** One-line mention of x402 + World ID

### 3. Directory Cards
- Logo-dense like Stackshare
- Clear attribution (who submitted, when)
- Minimal text per card — just essential info

### 4. Footer
- Remove duplicate branding from hero
- One clean line for x402 + World ID mention
- Clean links: About, API Docs, Submit, Contact

---

## Recommendations for Metered

1. **Rewrite hero headline** from "Metered — x402 service directory" to "The x402 API directory for AI agents and developers"
2. **Move all trust badges** (Coinbase, World Chain) from hero to footer
3. **Remove redundant World ID mentions** — one clear explanation in trust cards section
4. **Tighten footer** — remove duplicate branding, keep one clean x402 + World ID mention
5. **Directory cards** should be logo-dense, not text-dense
